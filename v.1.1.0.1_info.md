# 배달인(Baedalin) v.1.1.0.1 브랜치 상세 설명

이 브랜치는 배달 플랫폼 자동화 위젯의 핵심 기능인 **UI 분석 및 좌표 자동 인식**을 강화하는 데 초점을 맞추고 있습니다.

## 주요 업데이트 내역

### 1. 배민 위젯 좌표 자동 인식 구현
기존의 하드코딩된 좌표 방식에서 벗어나, Accessibility Service를 통해 화면상의 컨트롤을 직접 찾아내어 클릭하는 기능을 도입했습니다.
- **대상 앱**: 배달의민족 배달원용 앱 (`com.woowahan.bros`)
- **인식 규칙**:
    - **콜 확인**: `ai-mode-notification-item-0` ID 또는 "신규배달" 텍스트를 감지하여 팝업을 클릭합니다.
    - **배차 수락**: `신규배차_수락버튼` 설명을 가진 컨트롤을 찾아 정확한 중앙 지점을 클릭합니다.
    - **배차 거절**: `신규배차_거절버튼` 설명뿐만 아니라, "거절", "거절하기", "확인" 등의 텍스트를 가진 버튼을 모두 감지하여 거절 프로세스를 자동화합니다.
    - **경로 보기**: "경로", "지도", "길찾기" 등의 키워드를 포함한 컨트롤을 찾아내어 배달 동선을 확인할 수 있도록 지원합니다.
- **안전장치 (Fallback)**: 자동 인식에 실패할 경우, 사용자가 설정한 기존 좌표 또는 기본 프리셋 좌표로 즉시 대체하여 작동 중단을 방지합니다.

### 2. UI 스냅샷 기능 고도화
UI 분석의 정확도를 높이기 위해 덤프 데이터의 상세 속성을 대폭 추가했습니다.
- **추가된 속성**: Resource ID, 활성화 여부(Enabled), 포커스 가능 여부(Focusable), 스크롤 가능 여부(Scrollable), 체크 가능 여부(Checkable) 등.
- **용도**: 앱 업데이트로 인해 UI 구조가 변경되더라도, 강화된 속성값을 바탕으로 인식 로직을 빠르게 보정할 수 있습니다.

### 3. 키 이벤트 가로채기 로직 최적화
불필요한 시스템 리소스 소모를 줄이고 사용자 경험을 개선했습니다.
- 매핑된 키 이벤트 가로채기는 **매핑 활성화 상태** 및 **서비스가 지정된 앱(배민 등) 상단에 있을 때**만 작동하도록 필터링을 강화했습니다.
- 시스템 앱 및 본 앱(`baedalin`)에서는 키 필터링이 비활성화되어 일반적인 조작이 방해받지 않도록 개선했습니다.

## 기술적 변경 사항
- `KeyMapperAccessibilityService.kt`: `resolveDynamicCoordinate`, `findRectByCriteria` 함수 추가 및 `handleAction` 로직 수정.
- `MappingConfig.kt`: 하드코딩된 좌표 의존성 완화 준비.

## 향후 계획
- 쿠팡이츠 등 타 플랫폼 앱으로 자동 인식 로직 확장.
- 실제 배달 환경에서의 인식 성능 모니터링 및 규칙 고도화.
